FERRIER Enzo

Je me suis chargé du développement du site web dynamique ainsi que du script permettant de récupérer les données via le broker MQTT. J’ai intégré ces données dans une base MySQL et conçu leur affichage sur le site sous forme de tableaux comprenant des statistiques (minimum, maximum, moyenne). L’enregistrement des données a été automatisé à l’aide de crontab. Ce projet m’a permis de consolider mes compétences en développement web (HTML, CSS, PHP), en scripts bash, ainsi qu’en gestion de bases de données. J’ai particulièrement apprécié le lien entre la partie technique (capteurs, traitement) et la mise en forme accessible via une interface utilisateur fonctionnelle.

POLIDORO Kevin

J’ai pris en charge la conception de la base de données via PhpMyAdmin. J’ai créé les tables nécessaires au bon fonctionnement du site, en suivant les exigences du cahier des charges. J’ai également mis en place un tableau de bord Grafana connecté à InfluxDB pour afficher en temps réel les mesures des capteurs. Cette mission m’a permis de mieux saisir la relation entre les bases de données relationnelles et les outils de visualisation de données, ainsi que l’importance d’un schéma de données bien pensé dans un projet connecté.

MOHAMED Kevin

Ma mission principale était axée sur la gestion du projet. J’ai réalisé un diagramme de Gantt et organisé les tâches sur Trello afin de répartir clairement le travail entre les membres de l’équipe. Cette expérience m’a permis de me familiariser concrètement avec la planification, la répartition des rôles, le suivi de l’avancement, et l’adaptation face aux imprévus techniques. Elle m’a donné une meilleure compréhension de l’importance d’une organisation rigoureuse dans le cadre d’un projet informatique collaboratif.
